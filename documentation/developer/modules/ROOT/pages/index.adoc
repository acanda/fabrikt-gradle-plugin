:description: Developer documentation for the Fabrikt Gradle Plugin

Welcome to the Fabrikt Gradle Plugin developer documentation. This guide is intended for developers who want to contribute to the plugin or understand its internal architecture.

== Introduction

The Fabrikt Gradle Plugin integrates Fabrikt into Gradle builds, allowing users to generate Kotlin code from OpenAPI specifications. This developer guide provides information about the plugin's architecture, development setup, and contribution guidelines.

== Getting Started

* xref:development-setup.adoc[Development Setup] - Set up your development environment

== Project Structure

The project follows a standard Gradle plugin structure:

* `src/main/kotlin` - The main source code for the plugin
* `src/main/schema` - Schema definitions used for code generation
* `src/test/kotlin` - Test cases for the plugin
* `buildSrc` - Custom build logic for the plugin itself

== Key Components

The plugin consists of several key components:

* `FabriktPlugin` - The main plugin class that integrates with Gradle
* `FabriktExtension` - The extension that provides the DSL for configuring the plugin
* `FabriktTask` - The task that generates code from OpenAPI specifications
* Configuration model - Classes that represent the plugin's configuration options
